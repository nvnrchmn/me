# Implementasi Fase 5 - Superadmin Portal & Integrasi MySQL

## Dikerjakan
- **Prisma ORM Setup**: Mengonfigurasi `schema.prisma` dengan koneksi ke *database* MySQL (`sbdigita_nova`). Menentukan struktur data *User, Profile, Experience, SkillCategory, SkillItem, Project, Setting*.
- **Database Seeding**: Membuat skrip *seed* (`scripts/seed.ts`) yang membaca data dari JSON di `src/data/*` lalu mengisinya ke dalam *database* MySQL baru. Kredensial awal (admin) juga ditambahkan.
- **Refactoring API Global**: Mengubah file `src/lib/api.ts` agar mengambil data langsung menggunakan instansiasi `PrismaClient` (sebelumnya `fs` atau `import json`), bukan lagi berkas JSON statis. Tipe data seperti `JsonValue` dari `techStack` dan `highlights` dikonversi agar aman (*type-safe*) bagi UI.
- **Otentikasi**: Mengonfigurasi modul NextAuth.js (`src/lib/auth.ts` & Route Handlers) dengan tipe *CredentialsProvider* (Email & Password) dibantu *library* `bcryptjs` untuk mengamankan kata sandi. 
- **Admin Layout**: Membuat halaman *Login* (`/login`) dan halaman utama ber-URL terlindungi (`/admin/layout.tsx` & `/admin/page.tsx`). Halaman admin dibungkus oleh perlindungan `getServerSession`. Ada juga tombol *Sign Out* yang mengakhiri sesi aktif.

## Diasumsikan
- Untuk saat ini *database* berlokasi di lokal dengan server *Laragon* (port default 3306), dan user MySQL adalah `root` tanpa *password*.
- Parameter UI di *frontend* masih menganggap seluruh struktur data yang disuap oleh Prisma identik dengan respons `api.ts` terdahulu. Struktur ini berhasil divalidasi lewat Type Checking saat *Build*.

## Belum Diverifikasi
- Form Edit/Simpan CRUD spesifik untuk masing-masing panel Admin (`/admin/projects`, `/admin/experience`, dll) belum dibuat secara keseluruhan. Fokus iterasi ini adalah peletakan batu fondasi (Infrastruktur DB, Skema, Migrasi Awal, NextAuth, dan Layout Superadmin).
