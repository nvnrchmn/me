# Implementasi Fase 4 - CV & Contact Form

## Dikerjakan
- **Integrasi Data CV**: Memperbarui antarmuka (*interface*) API `Profile` (`src/lib/api.ts`) dan data lokal (`src/data/profile.json`) dengan menambahkan properti `cvUrl`. Saat ini diposisikan ke placeholder `/dummy-cv.pdf` sebagai jembatan sebelum fitur *upload file* CMS di Fase 5 terwujud.
- **Halaman CV (`/cv`)**: Membuat halaman khusus yang menarik data *Summary*, *Experience*, dan *Skills* menjadi satu tampilan memanjang (*scrollable*) dengan *header* yang terpusat. Terdapat tautan ke file statis yang diunggah (`cvUrl`) dengan ikon unduh.
- **Formulir Kontak (`/contact`)**: Membuat formulir interaktif di sisi klien (Client Component) menggunakan `react-hook-form`. Melibatkan skema `zod` untuk validasi `name` (min 2 karakter), `email` (format surel), dan `message` (min 10 karakter). Dilengkapi *loading state* dan keterangan notifikasi (sukses/gagal).
- **Pengirim Email SMTP (`/api/contact`)**: Mengintegrasikan `nodemailer` di dalam **Next.js Route Handler** (metode `POST`) yang membaca *environment variables* `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, dan dukungan `SMTP_SECURE` untuk opsi pengiriman *Transport Layer Security* (TLS). API ini juga melakukan validasi *double-check* di *server-side* menggunakan Zod.

## Diasumsikan
- Ketika `process.env.SMTP_USER` tidak ditemukan di server (karena belum dipasok di `.env` lokal atau di antarmuka Admin nanti), *Route Handler* hanya akan mencetak isi *email payload* ke konsol *server* (`console.log`) alih-alih melempar *Error*. Hal ini untuk memastikan *development* lokal tidak terhambat (*mocking success*).
- File PDF (`dummy-cv.pdf`) diasumsikan belum ada, sehingga tautan mungkin akan mengarah ke *404* sampai admin nanti mengunggah filenya.

## Belum Diverifikasi
- Pengiriman surel *real* dari *Nodemailer* via *provider* khusus seperti Resend SMTP atau Gmail App Password belum diverifikasi karena belum ada kredensial (*credentials*) asli yang dipasang di `.env` lokal.
