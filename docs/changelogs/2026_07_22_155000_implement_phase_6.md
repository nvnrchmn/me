# Implementasi Fase 6 - Modul CRUD Superadmin

## Dikerjakan
- **Page Transitions**: Menambahkan `src/app/template.tsx` menggunakan `framer-motion` (`AnimatePresence`-style entrance) agar setiap perpindahan rute, baik di halaman utama publik maupun di dalam dasbor Admin, memicu animasi mulus vertikal (Y-axis fade-in).
- **Server Actions Terpusat**: Membuat berkas aksi peladen `src/app/actions/admin.ts`. Seluruh operasi Create, Update, dan Delete untuk semua model (Profile, Experience, Project, Skill) dieksekusi di sini. Aksi ini juga dilindungi dengan pengecekan `getServerSession` mutlak agar tak bisa ditembus entitas liar.
- **Profile Module (`/admin/profile`)**: Membangun UI Form *Client-side* untuk mengedit data diri, statistik, dan tautan sosial media, yang jika disimpan, langsung merefleksikannya ke halaman utama (*revalidatePath*).
- **Experience Module (`/admin/experience`)**: Membangun modul kelola pengalaman kerja dengan mekanisme *list-view* serta form yang bisa ditukar statusnya (Create/Edit). Fitur array seperti *Highlights* (poin-poin pekerjaan) dikelola dengan tombol tambah/hapus *input* dinamis.
- **Projects Module (`/admin/projects`)**: Membangun daftar proyek. Form memfasilitasi pembuatan *Slug* otomatis dan array dinamis untuk penginputan *Tech Stack* satu-persatu.
- **Skills Module (`/admin/skills`)**: Mengadaptasi logika 2 tahap. Pertama, admin bisa menambahkan dan menghapus seluruh Kategori secara *inline*. Di bawah tiap Kategori tersebut, admin bisa langsung menekan "Add Skill Item" dan mengetikkan keahlian baru secara instan (menggunakan input *inline* agar ringkas).

## Diasumsikan
- Untuk modul Profil, hanya ada 1 entitas profil di database (`findFirst`). Saat pertama kali *load*, data tersebut di-*passing* dari Server ke Client untuk menjadi nilai awal (*defaultValue*) Form.
- Komponen transisi animasi `template.tsx` diletakkan di *root* `app/` agar berdampak global ke **seluruh** perpindahan *route*.

## Belum diverifikasi
- Mekanisme unggah (*Upload*) berkas fisik seperti unggah Avatar/Gambar Proyek ke direktori `public/` atau S3. Saat ini gambar masih *placeholder* (teks), karena pada struktur Prisma sebelumnya pun kita tidak menyentuh manajemen Storage.
- Modul *Settings* di panel samping (di luar 4 data inti) saat ini belum difungsikan penuh karena tak disebutkan eksklusif.
